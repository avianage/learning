# JVM Architecture — Quick Reference

## 3 Phases
**Loading** → **Execution** → **Garbage Collection**

## Memory Areas
| Area | Stores | Per |
|------|--------|-----|
| Method Area | Class metadata, static vars, constant pool | JVM (shared) |
| Heap | All `new` objects | JVM (shared) |
| Stack | Method frames, local vars, params | Thread |
| PC Register | Address of current instruction | Thread |
| Native Stack | Native (C/C++) calls via JNI | Thread |

## Class Loading
- **Bootstrap** → core Java (`java.lang.*`)
- **Extension** → `jre/lib/ext`
- **Application** → your classes/libs
- Loading is **lazy** (on first reference), not at startup.

## Heap Structure
```
Young Gen: Eden → Survivor S0 → Survivor S1
Old Gen:   long-lived / promoted objects
```
New objects → Eden. Survive GC cycles → promoted to Old Gen.

## Stack
- One per thread; LIFO frames per method call.
- Frame holds: local vars, params, intermediate values.
- Popped on method return.
- **StackOverflowError** = stack full (usually infinite recursion, no base case).

## Execution Engine
| Mode | Behavior |
|------|----------|
| Interpreter | Executes bytecode line-by-line; slow on repeats |
| JIT Compiler | Compiles "hot" methods → native code; near-native speed after warm-up |

## Garbage Collection
- Object eligible for GC when **unreachable** (no live reference).
- GC: mark reachable (from GC roots) → sweep unreachable → compact.
- `System.gc()` is a **request**, not a guarantee — avoid in production code.

| Collector | Default In | Best For |
|-----------|-----------|----------|
| Serial | — | Small/single-threaded |
| Parallel | Java 8 | Throughput |
| G1 | Java 9+ | Balanced latency/throughput |
| ZGC | Java 15+ | Sub-ms pause times |

## Stack vs Heap
| | Stack | Heap |
|---|---|---|
| Holds | Frames, locals, refs | Objects |
| Lifetime | Ends at method return | Until GC'd |
| Scope | Per-thread | Shared |
| Speed | Fast | Slower |
| Error | `StackOverflowError` | `OutOfMemoryError` |

## Common Errors
| Error | Cause |
|-------|-------|
| `StackOverflowError` | Infinite recursion |
| `OutOfMemoryError: Java heap space` | Heap full |
| `OutOfMemoryError: Metaspace` | Too many loaded classes |
| `ClassNotFoundException` | Class missing from classpath |
| `NoClassDefFoundError` | Class present at compile time, missing at runtime (JAR issue) |

## Key Flags
```bash
java -Xms256m -Xmx1g Employee   # initial 256MB, max 1GB heap
java -verbose:gc Employee       # log GC activity
```

## One-Line Memory Trace (mental model)
```
static var       → Method Area
new Object()      → Heap
local var / ref   → Stack (per thread frame)
```
