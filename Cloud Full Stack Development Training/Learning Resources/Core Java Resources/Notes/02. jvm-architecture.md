## Big Picture:

When you run `java Employee`, the JVM goes through three phases:

- Phase 1: Loading      -> Read Employee.class from disk --> load to memory
- Phase 2: Execution    -> Run the Bytecode instruction
- Phase 3: Cleanup      -> Garbage Collects objects no longer in use

JVM Memory Areas:
|-- Method Area     -> class definitions, static variables, constant
|-- Heap            -> all objects created with 'new'
|-- Stack           -> one per thread - method calls and local variables
|-- PC Register     -> one per thread - tracks current instrution
|-- Native Stack    -> for native (C/C++) method calls

We deal with Heap and Stack daily.
